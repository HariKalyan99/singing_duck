    🏗 How It Works In Practice
    Step 1 — During Deployment

    Before deploying, run:

    git rev-parse HEAD

    This prints:

    a3f9c1d8b72e41a6f...
    Step 2 — Set It As Environment Variable

    In your deployment platform:

    GIT_COMMIT=a3f9c1d8b72e41a6f...

    Now your server has access to:

    process.env.GIT_COMMIT
    Step 3 — Add It To captureDuck
    const errorObj = {
    ...
    commit: process.env.GIT_COMMIT || "unknown",
    };

    Now every error contains:

    {
    "message": "TypeError...",
    "commit": "a3f9c1d8b72e41a6f..."
    }
